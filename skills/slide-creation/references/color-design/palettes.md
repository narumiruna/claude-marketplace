# Ready-to-Use Color Palettes

This reference provides tested color palettes for different presentation contexts. Each palette includes hex values, usage guidance, and contrast validation.

## Dark Technical Palettes

### Palette 1: Code-Focused Blue

- **Background:** `#1E1E1E` — Main slide background (VS Code dark)
- **Surface:** `#252526` — Code blocks, diagram boxes
- **Primary:** `#569CD6` — Titles, headings (VS Code blue)
- **Secondary:** `#4EC9B0` — Section dividers, icons (cyan)
- **Accent:** `#F4BF75` — Important callouts, highlights (amber)
- **Text Primary:** `#D4D4D4` — Body text
- **Text Secondary:** `#858585` — Captions, metadata

**Best for:** Code-heavy presentations, technical demos, system architecture
**Contrast:** Text Primary/Background = 8.2:1, Primary/Background = 4.8:1
**Notes:** Inspired by VS Code theme; familiar to developers

### Palette 2: Terminal Dark

- **Surface:** `#1A1A1A` — Code blocks, panels
- **Primary:** `#61AFEF` — Titles, headings (bright blue)

- **Accent:** `#E06C75` — Warnings, critical points (red)

## Usage Guidelines

### Choosing a Palette

1. **Start with context:** Dark Technical for code/diagrams, Light Professional for business, Accent-Driven for storytelling
2. **Consider environment:** Projectors benefit from higher contrast (Terminal Dark, Clean Corporate)
3. **Match brand if applicable:** Adapt palette colors to incorporate brand colors while maintaining structure
4. **Test contrast:** Verify text remains legible in the actual presentation environment

### Customizing Palettes

- **Keep the structure:** Maintain Background/Surface/Primary/Secondary/Accent/Text roles
- **Adjust one color at a time:** Don't change multiple colors simultaneously
- **Preserve contrast ratios:** Use a contrast checker when modifying colors
- **Test on projector:** Colors appear differently on screens vs projected

### Common Mistakes to Avoid

- **Too many colors:** Stick to 5-7 colors maximum (the palettes above are complete)
- **Low contrast:** Avoid text that blends with background
- **Inconsistent application:** Once chosen, use the palette throughout all slides
- **Ignoring semantic meaning:** Don't use red/green together (colorblind issues)
